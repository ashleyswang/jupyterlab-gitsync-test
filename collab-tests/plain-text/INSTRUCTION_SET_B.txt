PART II: INTEGRATED EDITS W/O CONFLICTS

Instructions: In the template, there are 10 sets of 5 binary digits. 
For each set, given the attributes of the set listed by Tester A, 
your goal is to determine whether or not the set is a GOOD or BAD set.

1. Wait for the list of attributes from Tester A. The possible 
attributes are [red, blue, star, light]. The list should look something 
like this: 

    01001  => [red, blue, star] 

2. Then using the following tables, determine whether or not the set 
is GOOD or BAD. Once you have determined if the set is GOOD or BAD,
replace the attribute list given with your answer.

Syntax Notes:
- sum(digits): sum of all digits in the set
- digits[i]  : digit at ith index starting at 0; negative indexes are
               counted from the back with the last digit index at -1
- ternary    : (cond) ? (value if cond is true) : (value if cond is false)
  operator 
- boolean    : 0 is false, 1 is true

  ____________________________________________________________________
  --------------------------------------------------------------------
           |          red          |         !red          |        
  --------------------------------------------------------------------
           |     G     |     L     |     L     |     B     |  star
    blue   -----------------------------------------------------------
           |     M     |     M     |     L     |     M     | !star
  --------------------------------------------------------------------
           |     C     |     L     |     C     |     C     |  star
   !blue   -----------------------------------------------------------
           |     C     |     M     |     L     |     G     | !star
  --------------------------------------------------------------------
           |   light   |  !light   |   light   |  !light   |
  --------------------------------------------------------------------

  ____________________________________________________________________
  --------------------------------------------------------------------
    code   |  return value       
  --------------------------------------------------------------------
      B    |  BAD
  --------------------------------------------------------------------
      G    |  GOOD
  --------------------------------------------------------------------
      C    |  (sum(digits) > 2) ? GOOD : BAD
  --------------------------------------------------------------------
      M    |  (sum(digits) % 2) ? GOOD : BAD
  --------------------------------------------------------------------
      L    |  (digits[-1]) ? GOOD : BAD
  --------------------------------------------------------------------
  ¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯¯

EXAMPLE: 01001  => [red, blue, star] 

    1. This set has the red, blue, and star attribute. Looking at the 
       the first table we see this correlates to code L. 
    2. Since code L checks the last digit before returning a value, we
       check digits[-1]==1 which evaluates to GOOD. We then replace the
       attribute list with our answer.
       
       01001  => GOOD
